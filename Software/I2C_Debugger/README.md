This directory contains the monitoring application for the Modular Factory System.

The application monitors I²C-connected nodes, displays their current state, records state transitions, and collects power monitoring data.

---

## Purpose

The I2C Debugger is used to observe and debug the distributed I²C node system.

It helps confirm that motor, servo, sensor, and power monitoring boards are communicating correctly during tabletop demonstrations.

This program is optimized for running on a Raspberry Pi 5, which controls the entire system.

Although it can be started on a general PC in mock mode, its main purpose is to monitor the actual I²C bus and hardware state on the Raspberry Pi 5.

---

## Main Features

- monitors I²C node addresses from `0x10` to `0x19`
- displays node state such as READY, BUSY, WARN, ERROR, ESTOP, Signal Lost, and No Device
- decodes status bits and command values
- records state transitions to SQLite
- records periodic node snapshots
- records power monitor values from INA219 sensors
- supports real I²C mode and mock mode
- provides a PyQt5-based desktop UI
- stores session-based monitoring logs for later analysis

## Node Monitoring

The monitor reads I²C nodes in the configured address range.

Default address range : `0x10 - 0x19`

The following states are displayed:

| State | Meaning |
|---|---|
| READY | Node is ready |
| BUSY | Node is executing an operation |
| WARN | Warning state |
| ERROR | Error state |
| E-STOP | Emergency stop state |
| Signal Lost | Device was seen before, but communication was lost |
| No Device | No device detected |
| UNKNOWN | Temporary or undefined state |

---

## Power Monitoring

The application also collects power monitor data from INA219 sensors through TCA9548A I²C multiplexers.

The power monitoring layer records:

- port ID
- TCA address and channel
- INA219 address
- bus voltage
- shunt raw value
- current
- power
- session ID

The conversion from INA219 raw values is implemented in `power_calc.py`.

---

## Logging

Logs are stored in SQLite.

Default database path : `/var/lib/i2c_debugger/i2c_debugger_events.sqlite3`

The logger creates tables for:

- `event_logs`
- `node_snapshots`
- `run_sessions`
- `state_segments`
- `monitor_sessions`
- `power_port_snapshots`

These tables are used by the application itself and by Grafana dashboards.

---

## Reader Modes

The application supports two reader modes.

| Mode | Description |
|---|---|
| real | Reads actual I²C devices through SMBus |
| mock | Simulates node states and power values for UI / logic testing |

Mock mode is useful for checking UI behavior without connecting physical hardware.

---

## Launch

The launcher (run_i2c_debugger.sh) activates the Python virtual environment and starts the PyQt application.

The launcher script contains placeholder paths.  

Please update `APP_DIR` to match the directory where `I2C_Debugger` is placed on your Raspberry Pi 5.

---

## Requirements

- Python 3
- PyQt5
- smbus2
- sqlite3 (Python standard library)
  
---

## Relationship to Grafana

Grafana is not used for hardware control.

It reads the SQLite database generated by this monitoring application and visualizes node status, sessions, events, and power measurements.
