# Grafana

This directory contains notes for visualizing the SQLite logs generated by the I2C Debugger.

Grafana is used as a visualization layer, not as a hardware control layer.

---

## Purpose

Grafana displays:

- node state timelines
- recent events
- error and warning logs
- monitoring sessions
- power monitor values
- current / voltage / power trends

---

## Data Source

The dashboard uses SQLite as its data source.

The database is generated by the monitoring software: `/var/lib/i2c_debugger/i2c_debugger_events.sqlite3`

---

## Data Flow

```text
I2C Debugger
      ↓
SQLite database
      ↓
Grafana dashboard
```

Grafana reads the SQLite database created by the I2C Debugger and visualizes the recorded node states, events, sessions, and power monitor values.

---

## Dashboard Panels

The dashboard can include panels such as:

| Panel | Purpose |
|---|---|
|State Timeline | Shows node state transitions over time |
|Latest Events | Lists recent node state changes |
|Error / Warning Logs | Highlights ERROR, WARN, and ESTOP events |
|Monitor Sessions | Shows monitoring session start and end information |
|Power Monitor | Displays current, voltage, and power values for each port |

---

## Notes

Most Grafana-related documentation in this project consists of:

- SQL queries
- dashboard screenshots
- panel configuration notes
- time-range and session-filter notes

Grafana is used only for visualization and analysis.

Hardware control is performed by the host-side software and firmware nodes.
