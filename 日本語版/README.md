> 分散I²Cノード、独自基板、実アクチュエータ、ログ収集、Grafana可視化を組み合わせた、再構成可能な卓上ミニチュア生産ラインです。

*卓上ミニチュア生産ラインの使用例。コンベアでのワーク搬送、カメラでの識別など分散制御基板を含み動作します。*

---

## 概要

このプロジェクトは、工場DXや小規模な生産ラインの検証を想定した、卓上サイズのモジュール式ファクトリーシステムです。

Raspberry Pi 5を中核に、Raspberry Pi Picoベースの分散I²Cノード、独自設計基板、DCモータ、サーボモータ、センサ、カメラ、SQLiteログ、Grafana可視化を組み合わせています。

単にモータを動かすだけではなく、実際の工程に近い形で、

- センサで状態を取得する
- アクチュエータを制御する
- 状態遷移をログに残す
- 電流・電圧・電力を監視する
- Grafanaで可視化する
- 用途に応じて工程を組み替える

ところまでを一つの卓上環境で確認できることを目的としています。

---

## このプロジェクトで示したいこと

このシステムでは、以下のような要素を一体化しています。

- ハードウェア設計：独自PCB、I²C配線、電源監視基板
- 組み込み制御：Raspberry Pi Picoによる分散制御ノード
- 上位制御：Raspberry Pi 5による制御・監視・オーケストレーション
- センシング：フォトリフレクタ、カメラ画像認識
- ログ収集：SQLiteによる状態遷移・電力データ保存
- 可視化：Grafanaによるタイムライン・電力監視
- ユースケース：仕分け工程、スタンプ押印工程

これにより、「小さな装置を作った」だけではなく、制御・監視・記録・分析まで含めたDX的なシステム統合を、小規模な卓上環境で観察・実験できることを狙っています。

---

## 何を作ったのか

これは、工場や物流ラインで使われる制御・監視・可視化の考え方を、卓上サイズで再現するためのミニチュア生産ラインです。

ソフトウェアだけ、回路だけ、機構だけ、というように要素を分けて作るのではなく、最初から統合する前提で設計しています。


このプロジェクトでは、実際にすべての要素を取り込んで、以下を一つの流れとして実装しています。

```text
物理機構
  ↓
独自基板
  ↓
Pico側ファームウェア
  ↓
Raspberry Pi 5による上位制御
  ↓
SQLiteログ
  ↓
Grafana可視化
```

たとえば、Amazon風仕分けデモでは、コンベアで対象物を搬送し、センサで検知し、カメラで色を判定し、サーボゲートで仕分けます。

同じ制御基盤を使って、スタンプ押印デモでは、紙送り、紙押さえ、ラックピニオンによる押印という別工程に組み替えています。


このように、同じハードウェア・ファームウェア・ソフトウェア基盤を再利用しながら、異なる工程に展開できることを示しています。

---

## ポートフォリオとしてのポイント

このプロジェクトでは、実装にあたって以下の事柄をすべて対応しています。

- 実機を使った制御システムの設計・実装
- Raspberry Pi / Raspberry Pi Pico / I²C を用いた分散制御
- KiCad等による独自PCB設計と製造データ整理
- DigiKey MyListによるBOM管理
- Python / PyQtによる監視ソフト開発
- SQLiteによるログ保存
- Grafanaによる可視化
- センサ・カメラ・アクチュエータを組み合わせた工程設計
- 仕様変更や別工程への再構成を前提とした設計

単体の装置ではなく、制御・電装・ソフトウェア・データ可視化を横断したシステムとしてまとめています。
